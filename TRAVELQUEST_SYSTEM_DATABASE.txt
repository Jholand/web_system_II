PROJECT OVERVIEW
================================================================================
TravelQuest is a location-based gamification platform where users explore
destinations, earn points, collect badges, and redeem rewards. The system
includes an admin panel for managing destinations, badges, and rewards.

TECH STACK:
- Frontend: React 19.1.1 with Vite
- Routing: React Router v6
- Animations: Framer Motion
- Styling: Tailwind CSS
- Backend: Laravel (API-ready, in development)
- State Management: React Context API
- Notifications: React Hot Toast

The app uses a prefix-based routing system for clear separation:

PUBLIC ROUTES:
- /login                    ‚Üí User login page

- /user/dashboard           ‚Üí UserDashboard: View profile & stats
- /user/checkin             ‚Üí CheckIn: QR code scanning for check-ins
- /user/rewards             ‚Üí Rewards: Browse and redeem rewards
- /user/settings            ‚Üí UserSettings: Account settings

ADMIN ROUTES (Prefix: /admin/):
- /admin/dashboard          ‚Üí Dashboard: Overview statistics

- /admin/badges             ‚Üí Badges: Manage badge system
- /admin/rewards            ‚Üí Rewards: Manage reward catalog
- /admin/map                ‚Üí AdminMap: View destinations on map

LEGACY REDIRECTS (Backward Compatibility):
- /map ‚Üí /user/map
- /dashboard ‚Üí /user/dashboard
- /checkin ‚Üí /user/checkin

4. Each route change triggers page-level animations


================================================================================
                    CONTEXT API SYSTEM
================================================================================

1. AUTH CONTEXT (AuthContext.jsx)
================================================================================
PURPOSE: Manages user authentication state globally

STATE VARIABLES:
- user: Current user object (name, email, role, etc.)
- loading: Boolean for initial auth check
- isAuthenticated: Boolean for auth status

FUNCTIONS:
- login(credentials): Authenticates user with API
  * Stores auth token in localStorage
  * Sets user state
  * Returns {success: true/false, user/error}

- register(userData): Creates new user account
  * Calls API registration endpoint
  * Auto-logs in after successful registration

- logout(): Clears user session
  * Removes token from localStorage
  * Resets user state
  * Redirects to login

- checkAuth(): Runs on app mount
  * Checks for existing token
  * Fetches user data if token exists
  * Sets authenticated state

USAGE:
  const { user, login, logout, isAuthenticated } = useAuth();


2. LOCATION CONTEXT (LocationContext.jsx)
================================================================================
PURPOSE: Handles geolocation tracking for proximity features

STATE VARIABLES:
- userLocation: {latitude, longitude, accuracy}
- locationError: Error message if tracking fails
- isTracking: Boolean for tracking status
- watchId: ID for geolocation watch session

FUNCTIONS:
- startTracking(): Initiates continuous location tracking
  * Uses navigator.geolocation.watchPosition()
  * Updates location in real-time
  * Handles permission errors

- stopTracking(): Stops location tracking
  * Clears watch session
  * Called on component unmount

- calculateDistance(lat1, lon1, lat2, lon2): Haversine formula
  * Returns distance in kilometers
  * Used for nearby destination detection

- getNearbyDestinations(destinations, radius): Filters destinations
  * Returns destinations within specified radius
  * Uses calculateDistance() for each destination

USAGE:
  const { userLocation, getNearbyDestinations } = useLocation();

TOKEN MANAGEMENT:

- getHeaders(): Returns authorization headers with Bearer token
- Token automatically included in all authenticated requests


================================================================================
                    ANIMATION SYSTEM
================================================================================

FILE: utils/animations.js

PURPOSE: Centralized Framer Motion animation variants for consistency

ANIMATION STRATEGY:
- High initial opacity (0.95-0.98) prevents black flash
- Short durations (0.15-0.2s) for snappy feel
- Easing functions for natural motion
- Staggered animations for sequential reveals

AVAILABLE VARIANTS:

1. fadeIn
   - hidden: opacity 0.95
   - visible: opacity 1 (0.15s)
   - exit: opacity 0.95 (0.1s)
   USE CASE: Subtle content reveals

2. slideInFromRight
   - hidden: {opacity: 0.95, x: 20}
   - visible: {opacity: 1, x: 0} (0.2s)
   USE CASE: Sidebar content, cards from right

3. slideInFromLeft
   - hidden: {opacity: 0.95, x: -20}
   - visible: {opacity: 1, x: 0} (0.2s)
   USE CASE: Content panels, lists

4. slideInFromBottom
   - hidden: {opacity: 0.95, y: 20}
   - visible: {opacity: 1, y: 0} (0.2s)
   USE CASE: Modal entry, bottom sheets

5. staggerContainer
   - hidden: opacity 1
   - visible: opacity 1, staggerChildren: 0.08s, delayChildren: 0.05s
   USE CASE: Wrap around multiple animated children

6. scaleIn
   - hidden: {scale: 0.95, opacity: 0.95}
   - visible: {scale: 1, opacity: 1} (0.2s spring)
   USE CASE: Cards, important elements

7. modalVariants
   - hidden: {scale: 0.98, y: 10, opacity: 0}
   - visible: {scale: 1, y: 0, opacity: 1} (0.2s)
   USE CASE: Modal dialogs, popups

8. tabSlide
   - hidden: {x: ¬±50, opacity: 0.95}
   - visible: {x: 0, opacity: 1} (0.2s)
   USE CASE: Tab content switching

USAGE EXAMPLE:
  <motion.div
    variants={slideInFromRight}
    initial="hidden"
    animate="visible"
  >
    Content here
  </motion.div>

5. Scanner detects and decodes QR
6. Validates destination exists
7. Checks user location (must be within 100m)
8. Verifies not checked in recently (cooldown)
9. Creates check-in record
10. Awards points to user

1. USER DISCOVERS DESTINATION:

   - Redirected to /user/checkin OR
   - Modal opens with QR scanner

4. QR CODE SCANNING:
   - Camera activates
   - User scans QR code at location
   - QR decoded: {destinationId: 1, timestamp, signature}

5. VALIDATION (Frontend):
   - Checks QR format is valid
   - Verifies signature hasn't expired
   - Gets user location from LocationContext

6. API REQUEST:
   POST /api/checkins
   Body: {
     destinationId: 1,
     qrCode: "scanned_code",
     userLocation: {
       latitude: 14.5995,
       longitude: 120.9842,
       accuracy: 10
     },
     timestamp: "2024-01-15T10:30:00Z"
   }

7. BACKEND VALIDATION:
   - Verifies user is authenticated
   - Checks destination exists and is active
   - Calculates distance (user to destination)
   - Validates distance < 100m
   - Checks user hasn't checked in today
   - Verifies QR signature
   - Checks QR hasn't expired

8. POINTS CALCULATION:
   basePoints = destination.points (50)
   firstVisitBonus = isFirstVisit ? 25 : 0
   streakBonus = userStreak * 10
   totalPoints = basePoints + bonuses

9. DATABASE UPDATES:
   - Create check_in record
   - Update user.points += totalPoints
   - Update user.total_visits++
   - Update user.current_streak
   - Update destination.visit_count++
   - Check badge requirements
   - Award badges if requirements met

10. API RESPONSE:
    {
      success: true,
      message: "Check-in successful!",
      points_earned: 85,
      total_points: 2535,
      badges_unlocked: ["Explorer"],
      destination: {
        name: "Grand Hotel Resort",
        category: "hotel"
      }
    }

    - Confetti animation

AUTHENTICATION SECURITY:
================================================================================
- JWT tokens for API authentication
- Token stored in localStorage (secure context only)
- Token expiration: 24 hours
- Refresh token mechanism
- HTTPS required in production
- CORS configured for frontend domain only

PASSWORD SECURITY:
- Bcrypt hashing (10 rounds minimum)
- Password requirements enforced
- Password strength indicator
- No password in API responses
- Password reset via email token
- Token expires after 1 hour

QR CODE SECURITY:
- Digital signature using HMAC-SHA256
- Timestamp validation (max 5 minutes old)
- One-time use codes
- Location verification required
- Rate limiting (max 10 scans per minute)
- Invalid attempts logged

LOCATION SECURITY:
- GPS accuracy requirements (< 50m)
- Distance verification (server-side)
- Motion sensor validation (anti-spoofing)
- IP geolocation cross-check
- Suspicious activity detection
- Location history tracking

API SECURITY:
- Rate limiting per user/IP
- Input validation and sanitization
- SQL injection prevention (parameterized queries)
- XSS prevention (escaped output)
- CSRF tokens for state-changing operations
- API key rotation
- Request signing for critical operations

   - Check device performance
   - Disable other animations
   - Update browser

   - Hold steady
   - Clean camera lens
   - Try different angle
   - Check QR code is valid
